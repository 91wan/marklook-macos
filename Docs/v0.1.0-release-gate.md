# v0.1.0 Release Gate

## What Must Pass Before Tagging

- Latest `main` is checked out and clean.
- `Scripts/validate-v0.1.0-release-candidate.sh --ci` passes.
- Local Apple Development gate passes with `DEVELOPMENT_TEAM=<TEAM_ID> Scripts/validate-v0.1.0-release-candidate.sh --local`, or the owner explicitly defers local validation.
- PlugInKit lists only the installed `/Applications/MarkLook.app` MarkLook Preview/Thumbnail registrations after local validation.
- Latest-main package artifacts are regenerated and use the latest short SHA.
- Manual Finder preview acceptance is recorded or explicitly deferred by the owner.
- Known limitations are accepted.
- The owner explicitly approves pushing the `v0.1.0` tag after merge.

## Local-Only Artifacts

- Unsigned CI packages are build/package validation artifacts only.
- Apple Development packages are local validation artifacts only.
- Neither artifact proves public Gatekeeper trust.

## What The Tag Means

The `v0.1.0` tag marks a source/local-validation milestone unless Developer ID notarized artifacts are also available.

## What The Tag Does Not Mean

- It does not mean a public notarized binary exists.
- It does not mean Homebrew installation is available.
- It does not mean Gatekeeper trust has been proven for public distribution.

## Public Binary Trust

Do not publish a public binary release claiming Gatekeeper trust without Developer ID Application signing, notarization, stapling, and `spctl` assessment on the final artifact.

## Tag Commands

Use only after the PR merges, latest `main` passes final validation, and the owner approves tag creation:

```bash
git checkout main
git pull --ff-only
git rev-parse HEAD
Scripts/validate-v0.1.0-release-candidate.sh --ci
DEVELOPMENT_TEAM=<TEAM_ID> Scripts/validate-v0.1.0-release-candidate.sh --local
pluginkit -mADv | grep -i MarkLook
git tag -a v0.1.0 -m "MarkLook v0.1.0"
git push origin v0.1.0
```

Do not publish a GitHub Release automatically.
