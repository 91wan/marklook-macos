# v0.1.1 Source / Local Validation Gate

This document defines the v0.1.1 source/local-validation patch boundary.

## Scope

v0.1.1 is intended to mark post-v0.1.0 source and local-validation fixes that are already on `main` or queued for this patch:

- Deterministic Markdown thumbnail rendering.
- Non-overlapping thumbnail title and MD badge layout.
- Current-tree public privacy cleanup.
- Permanent public repository privacy gate for current and archived scans.
- Documented fixed light thumbnail appearance policy for v0.1.x.
- Chinese README page.

## Non-goals

This patch does not:

- Create or push the `v0.1.1` tag from a PR branch.
- Publish a GitHub Release.
- Submit or update a Homebrew cask.
- Claim Developer ID, notarization, or stapling trust.
- Rewrite Git history.

## Required Before Tag Approval

Run these checks from the final merged commit on latest `main`:

```bash
git status --short
Scripts/validate-public-repo-privacy.sh
Scripts/validate-public-repo-privacy.sh --archive
xcodegen generate
Scripts/validate-v0.1.0-release-candidate.sh --ci
```

Run the maintainer Mac local gate when an Apple Development identity is available:

```bash
Scripts/doctor-signing.sh
DEVELOPMENT_TEAM=<TEAM_ID> Scripts/validate-v0.1.0-release-candidate.sh --local
```

If local Apple Development validation is unavailable, record the exact reason in the PR before requesting tag approval.

The existing gate script is still named `validate-v0.1.0-release-candidate.sh`; for v0.1.1 it is reused as the established CI/local validation gate.

## Bundle Version

v0.1.1 requires:

```text
CFBundleShortVersionString = 0.1.1
CFBundleVersion = 2
```

for:

- MarkLook.app
- MarkLookPreview.appex
- MarkLookThumbnail.appex

## Tag Boundary

The tag is allowed only after the owner gives a second explicit approval after merge and final latest-main validation.

Approved tag commands, when that approval exists:

```bash
git tag -a v0.1.1 -m "MarkLook v0.1.1"
git push origin v0.1.1
```
